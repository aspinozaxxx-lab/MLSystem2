# MLSystem 2.0

MLSystem 2.0 — минимальная система обучения и инференса, запускаемая из командной строки.
Имя пакета — `mlsystem2`, чтобы проект не конфликтовал со старым MLSystem.

В репозитории остается только необходимое и достаточное: типизированные настройки, слой доступа к
хранилищу, контракты подготовки датасета, контракты источников тайлов, контракты моделей и
чекпойнтов, метрики, оркестрация обучения, оркестрация инференса и один адаптер MLflow.

## Отличие от старого MLSystem

Старый MLSystem можно использовать только как источник идей и доменной логики для следующих шагов
миграции. Инфраструктура из старого проекта сюда не переносится.

В этом репозитории намеренно нет ФастАПИ, Ювикорн, Эйрфлоу, Рэббит-эм-кью, Тритон-сервера,
Докера, Докер-компоуза, Ансибла, Графаны, Прометея, фронтенда, реестра заданий, реестра стадий,
устаревших оберток и совместимых слоев для старого кода.

## Запуск

```bash
pip install -e .[dev]
mlsystem2-train --config configs/example.local.yaml
mlsystem2-infer --config configs/example.local.yaml
```

Каноническое место хранения результатов обучения и инференса — MLflow. Локальные директории
используются только как временная рабочая область и не должны становиться источником истины для
результатов.

Серверная доставка является только копированием кода. Процесс CD копирует репозиторий на сервер
через SSH и записывает идентификатор доставленного коммита. Если целевой сервер — `31.192.104.147`,
его адрес можно указать в секретах GitHub.
