# Архитектура

MLSystem 2.0 — простая система командной строки с явными границами модулей.

Основное приложение обучения:

```bash
mlsystem2-train --config configs/example.server.yaml
```

Основное приложение инференса:

```bash
mlsystem2-infer --config configs/example.server.yaml
```

Каноническое место хранения результатов работы — MLflow. Локальные директории используются только
как временная рабочая область: в них могут лежать временные файлы, кэши, журналы и промежуточные
отчеты до записи в артефакты MLflow.

## Конвейер Обучения

1. Загрузить настройки.
2. Создать или открыть запуск MLflow через `mlflow_adapter`.
3. `dataset_preparing` проверяет наличие снимков и готовит разбиение и footprints.
4. Если `dataset_preparing` вернул ошибки, `train_pipeline` записывает отчет подготовки в MLflow и
   завершает конвейер с ошибкой.
5. `tile_preparation` создает источники train/val тайлов.
6. `train` обучает модель.
7. `mlflow_adapter` записывает метрики, артефакты, модель или чекпойнт, отчет времени и итоговый
   отчет.

## Конвейер Инференса

1. Загрузить настройки.
2. Загрузить модель или чекпойнт.
3. Выполнить инференс напрямую в Python/PyTorch.
4. Записать результаты и отчеты в MLflow.

## Явно Вне Области Проекта

В MLSystem 2.0 нет ФастАПИ, Эйрфлоу, Рэббит-эм-кью, Тритон-сервера, Докера, Ансибла, Графаны,
Прометея, фронтенда, реестра заданий, реестра стадий, устаревших оберток и артефактов
совместимости.
