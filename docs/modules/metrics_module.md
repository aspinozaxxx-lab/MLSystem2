# Модуль metrics

## Назначение

`metrics` считает минимальный набор метрик. Основная публичная метрика обучения — pixel F1.

## Публичный интерфейс

- `compute_pixel_f1(request: PixelF1Request) -> PixelF1Result`
- `summarize_epoch_metrics(history: list[EpochMetrics]) -> MetricsSummary`

Параметры:

- `request.y_true`: бинарная эталонная маска пикселей.
- `request.y_pred`: бинарная предсказанная маска или значения оценки.
- `request.threshold`: порог для значений оценки.
- `history`: записи метрик по эпохам.

## Контракты

`PixelF1Request`, `PixelF1Result`, `EpochMetrics`, `MetricsSummary`, `MetricsError`.

## Выходные артефакты

Прямых артефактов нет. Сводки могут логировать модули конвейеров или адаптеры.

## Что модуль НЕ делает

Не пишет в MLflow, не принимает решения конвейера и не создает модель.

## Разрешенные зависимости

NumPy и стандартная библиотека math.

## Запрещенные пересечения

Не импортирует внутренности конвейера, моделей и адаптера MLflow.

## MLflow

Напрямую не участвует.

## Временное профилирование

Время не замеряет.
