# Модуль inference_pipeline

## Назначение

`inference_pipeline` оркестрирует приложение инференса из командной строки: загрузку настроек,
вызов инференса, логирование в MLflow через адаптер, замеры времени и сборку результата.

## Публичный интерфейс

`run_inference_pipeline(request: InferencePipelineRequest) -> InferencePipelineResult`

Параметры:

- `request.config_path`: путь к YAML-конфигу.
- `request.run_name`: необязательное имя запуска MLflow.

## Контракты

`InferencePipelineRequest`, `InferencePipelineResult`, `InferencePipelineError`.

## Выходные артефакты

Ссылки на результат инференса, ссылка на запуск MLflow, объекты замера времени и отчета.

## Что модуль НЕ делает

Не импортирует MLflow напрямую, не реализует алгоритм инференса, не обучает и не управляет
жизненным циклом сервисов.

## Разрешенные зависимости

Публичные API настроек, инференса и адаптера MLflow.

## Запрещенные пересечения

Не импортирует `_*.py` файлы других модулей.

## MLflow

Использует только `mlflow_adapter.api`.

## Временное профилирование

Владеет замерами времени инференса на уровне конвейера.
