# Модуль inference

## Назначение

`inference` выполняет прямой Python/PyTorch инференс без сервисной инфраструктуры. Модуль
загружает модель или чекпойнт через публичные API моделей, читает снимки через публичные API
storage, выполняет инференс по тайлам или окнам и возвращает `InferenceResult`.

## Публичный интерфейс

`run_inference(request: InferenceRequest) -> InferenceResult`

Параметры:

- `request.config`: конфигурация инференса.
- `request.images_uri`: URI исходных снимков.
- `request.output_uri`: URI выходных артефактов.
- `request.model_spec`: необязательная спецификация модели.

## Контракты

`InferenceRequest`, `InferenceResult`, `InferenceConfig`, `InferenceArtifact`, `InferenceError`.

## Выходные артефакты

Артефакты инференса: маски, векторы или отчеты, представленные через `InferenceArtifact`.

## Что модуль НЕ делает

Не пишет в MLflow, не управляет жизненным циклом конвейера, не использует Тритон и не использует сервисные
очереди.

## Разрешенные зависимости

Необязательный `torch`, `models.api`, `storage.api` и приватные вспомогательные функции для тайлов
или окон.

## Запрещенные пересечения

Не импортирует внутренности конвейера инференса, адаптера MLflow и обучения.

## MLflow

Напрямую не участвует. Pipeline-модули логируют выходы инференса.

## Временное профилирование

Может вернуть замеры операций в своем отчете. Pipeline владеет общим замером времени.
