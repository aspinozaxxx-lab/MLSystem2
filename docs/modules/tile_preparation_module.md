# Модуль tile_preparation

## Назначение

`tile_preparation` превращает готовый `DatasetManifest` в ограниченные источники train/val батчей
тайлов. Модуль использует границы сцен для поиска соседних снимков, не заполняет соседние зоны
черным при наличии соседнего снимка, заранее готовит тайлы, держит ограниченную очередь и по умолчанию
использует 16 рабочих потоков.

## Публичный интерфейс

`build_tile_sources(request: TileSourceRequest) -> TileSourceBundle`

Параметры:

- `request.manifest`: готовый манифест датасета.
- `request.config`: размер тайла, stride, число рабочих потоков предварительной подготовки, размер
  очереди предварительной подготовки и режим использования границ снимков.
- `request.scratch_uri`: необязательная временная область для подготовленных тайлов.

## Контракты

`TileSourceRequest`, `TileSourceBundle`, `TileBatch`, `TileBatchSource`,
`TilePreparationConfig`, `TilePreparationReport`, `TilePreparationError`.

## Выходные артефакты

Временные подготовленные батчи тайлов и `TilePreparationReport`.

## Что модуль НЕ делает

Не обучает, не пишет в MLflow, не делает train/val разбиение, не создает модель и не импортирует
внутренности train.

## Разрешенные зависимости

Контракты датасета, публичный API storage, примитивы ограниченной очереди и потоков, а также
приватные геометрические или растровые вспомогательные функции.

## Запрещенные пересечения

Не импортирует внутренности train, конвейера обучения, моделей и адаптера MLflow.

## MLflow

Напрямую не участвует. Pipeline-модули могут логировать его отчет.

## Временное профилирование

`TileBatchSource.profile_snapshot()` может возвращать вложенные замеры времени и статистику очереди.
