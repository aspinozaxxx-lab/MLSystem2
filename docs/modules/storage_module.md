# Модуль storage

## Назначение

`storage` дает единый слой доступа к локальным и S3 URI: разрешение URI, проверку существования,
чтение и запись JSON, список файлов и открытие raster-источника на уровне абстракции.

## Публичный интерфейс

- `resolve_uri(uri: str) -> ResolvedUri`
- `exists(uri: str) -> bool`
- `list_files(uri: str, *, suffixes: tuple[str, ...] | None = None) -> list[StorageObject]`
- `read_json(uri: str) -> dict[str, object]`
- `write_json(uri: str, payload: dict[str, object]) -> None`

Параметры:

- `uri`: локальный путь, `file://` URI или `s3://` URI.
- `suffixes`: необязательные разрешенные суффиксы имен файлов при листинге.
- `payload`: JSON-совместимый словарь.

## Контракты

`ResolvedUri`, `StorageObject`, `StorageBackend`, `StorageError`.

## Выходные артефакты

JSON-файлы, записанные через `write_json`.

## Что модуль НЕ делает

Не знает о train/val, не считает границы снимков, не пишет в MLflow и не оркестрирует конвейер.

## Разрешенные зависимости

pathlib, json, ленивое использование S3-клиента и приватная реализация raster-источников.

## Запрещенные пересечения

Не импортирует подготовку датасета, обучение, конвейеры и их приватные файлы.

## MLflow

В MLflow не участвует.

## Временное профилирование

Не владеет замерами времени. Вызывающий код может замерять операции хранилища.
