# Модуль storage

## Назначение

`storage` содержит минимальный публичный слой операций с URI для существующих модулей. Текущий `dataset_preparing` его не использует и принимает локальные пути напрямую.

## Публичный интерфейс

- `resolve_uri(uri: str) -> ResolvedUri` — разбирает локальный или S3 URI.
- `exists(uri: str) -> bool` — проверяет наличие объекта.
- `list_files(uri: str, suffixes: tuple[str, ...] | None = None) -> list[StorageObject]` — возвращает файлы по URI.
- `read_json(uri: str) -> dict[str, object]` — читает JSON-объект.
- `write_json(uri: str, payload: dict[str, object]) -> None` — записывает JSON-объект.

## Публичные контракты

- `StorageError` — ошибка операции хранилища.
- `ResolvedUri` — поля `uri`, `scheme`, `path`, `bucket`, `key`.
- `StorageObject` — поля `uri`, `name`, `size_bytes`, `modified_at`.
- `StorageBackend` — протокол минимальных операций хранилища.

## Список используемых данным модулем модулей и с какой целью

Модуль не использует публичные API других модулей.

## Алгоритм работы и его особенности

Фасад выбирает локальную или S3-реализацию по префиксу `s3://` и делегирует операцию приватному backend. Новые операции не добавляются для подготовки датасета, потому что этот шаг сейчас работает только с локальными путями.
