# Модуль train_pipeline

## Назначение

`train_pipeline` оркестрирует обучение: загрузку настроек, жизненный цикл запуска MLflow, подготовку
датасета, создание источников тайлов, обучение, замеры времени и сборку итогового отчета.

## Публичный интерфейс

`run_train_pipeline(request: TrainPipelineRequest) -> TrainPipelineResult`

Параметры:

- `request.config_path`: путь к YAML-конфигу.
- `request.run_name`: необязательное имя запуска MLflow.

## Контракты

`TrainPipelineRequest`, `TrainPipelineResult`, `PipelineStatus`, `ModuleTiming`,
`TimingReport`, `PipelineReport`, `TrainPipelineError`.

## Выходные артефакты

Ссылка на запуск MLflow, отчет времени, итоговый отчет конвейера, отчет подготовки датасета,
метрики обучения и артефакты чекпойнта или модели.

## Что модуль НЕ делает

Не импортирует MLflow напрямую, не реализует алгоритмы тайлов, не реализует цикл обучения и не
реализует MLflow policy.

## Разрешенные зависимости

Публичные API и контракты настроек, подготовки датасета, подготовки тайлов, моделей, обучения и
адаптера MLflow.

## Запрещенные пересечения

Не импортирует `_*.py` файлы других модулей и не реализует их приватные алгоритмы.

## MLflow

Использует только `mlflow_adapter.api` для создания и завершения запусков, а также для логирования
отчетов, метрик и артефактов.

## Временное профилирование

Владеет замерами времени для подготовки датасета, инициализации подготовки тайлов, обучения,
логирования MLflow, общего времени конвейера и необязательных вложенных замеров из отчетов тайлов
или обучения.
