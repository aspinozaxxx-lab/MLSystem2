# Модуль settings

## Назначение

`settings` загружает единый YAML-конфиг, проверяет базовые пути и настройки, затем возвращает
типизированные настройки для остальных модулей.

## Публичный интерфейс

`load_settings(path: str | Path) -> SystemSettings`

Параметры:

- `path`: локальный путь к YAML-файлу настроек.

## Контракты

`SystemSettings`, `StorageSettings`, `DatasetSettings`, `TilePreparationSettings`,
`TrainSettings`, `InferenceSettings`, `MLflowSettings`, `RuntimeSettings`, `SettingsError`.

## Выходные артефакты

Артефакты не создаются. Результат работы — DTO `SystemSettings`.

## Что модуль НЕ делает

Не пишет в MLflow, не обучает модель, не выполняет инференс, не выполняет S3 ввод-вывод и не
создает рабочие директории без явного запроса.

## Разрешенные зависимости

Pydantic, PyYAML, pathlib.

## Запрещенные пересечения

Не импортирует обучение, инференс, подготовку датасета, подготовку тайлов, внутренности хранилища и
внутренности адаптера MLflow.

## MLflow

В MLflow не участвует.

## Временное профилирование

Время не замеряет. При необходимости загрузку настроек замеряют модули конвейеров.
